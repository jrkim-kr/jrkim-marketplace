# Term Glossary — 기술 용어 이중 언어 정리

## 목적

기술 면접 준비와 학습을 위해, 기술 용어를 **한국어 + 중국어** 이중 언어로 정리한다. architect-advisor의 횡단 레이어로 동작하기도 하고, 독립적으로 호출할 수도 있다.

## 호출 모드

| 모드 | 트리거 | 동작 |
|------|--------|------|
| **독립 호출** | `/term-glossary [용어들]` | 지정된 용어를 즉시 정리 → Notion DB 저장 |
| **대화 추출** | `/term-glossary` (인자 없이) | 현재 대화에서 기술 용어를 자동 추출 → Notion DB 저장 |
| **워크플로우 연동** | architect-advisor portfolio step 완료 시 | 워크플로우 중 등장한 모든 용어를 누적 정리 → Notion DB 저장 |

## DB 컬럼 스키마

| 컬럼명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| `용어` | title | 한국어 용어 (영어 금지, Glossary로 분리) | 멱등성 |
| `术语` | text | 중국어 용어 | 幂等性 |
| `Glossary` | text | 영어 용어 + 약어 | Idempotency |
| `발음` | text | IPA 발음 기호 | /ˌaɪ.dəmˈpoʊ.tən.si/ |
| `비유` | text | 한국어 일상 비유 | 엘리베이터 버튼을 10번 눌러도 한 번만 호출 |
| `정의` | text | 한국어 기술 정의 | 동일한 연산을 여러 번 수행해도 결과가 변하지 않는 성질 |
| `类比` | text | 중국어 일상 비유 | 电梯按钮按10次也只会响应一次 |
| `定义` | text | 중국어 기술 정의 | 同一操作执行多次，结果与执行一次完全相同的特性 |
| `Use Cases` | text | 실무 활용 시나리오 (쉼표 구분) | 결제 재시도, API 중복 호출 방지 |
| `프로젝트 적용 사례` | text | 프로젝트 맥락 적용 예시 | 동방텍스타일 — 결제 요청마다 고유 키 부여 |
| `카테고리` | select | 분류 | 데이터베이스 / 소프트웨어 개발 / 클라우드 & 인프라 / AI 및 기타 |

## 작성 규칙

### 용어 컬럼 규칙
- `용어`에는 **순수 한국어만** 작성 (영어 금지)
- 영어가 포함된 원래 이름은 `Glossary` 컬럼으로 분리
- 예: `용어`: "행 수준 보안 정책", `Glossary`: "Row Level Security (RLS)"

### 콘텐츠 작성 원칙
- **비유/类比**: 일상생활에서 가져온다 — 기술을 모르는 사람도 직관적으로 이해할 수 있어야 한다
- **정의/定义**: 기술적으로 정확하되 간결하게 1-2문장
- **Use Cases**: 일반적인 실무 활용 시나리오를 쉼표로 나열
- **프로젝트 적용 사례**: 현재 프로젝트(동방텍스타일, 에이사허브 등)나 대화 맥락에 연결
- 한국어와 중국어 블록은 직역이 아니라, 각 언어 화자에게 자연스럽게 읽히도록 작성
- **모든 컬럼은 반드시 채워야 한다** — 빈 셀 금지

### 카테고리 분류
| 카테고리 | 대상 |
|----------|------|
| 데이터베이스 | DB, SQL, 트랜잭션, 스키마, 인덱스 등 |
| 소프트웨어 개발 | 테스트, 미들웨어, 디자인 패턴, 프레임워크 등 |
| 클라우드 & 인프라 | 서버, 배포, 스케일링, IaaS/PaaS/BaaS 등 |
| AI 및 기타 | 벡터 검색, 임베딩, LLM, 기타 |

## 실행 절차

1. **용어 수집**: 인자로 전달된 용어 또는 현재 대화에서 기술 용어 추출
2. **중복 확인**: Notion DB에서 동일 용어가 이미 존재하는지 검색
3. **컨텐츠 생성**: 각 용어에 대해 11개 컬럼 데이터 생성
4. **Notion DB 저장**: `notion-create-pages` (신규) 또는 `notion-update-page` (기존) 사용
   - parent: `{ "type": "data_source_id", "data_source_id": "307ffa74-a2ce-801b-847c-000b85b30ba0" }`
5. **결과 보고**: 추가/수정된 용어 목록을 테이블로 출력
